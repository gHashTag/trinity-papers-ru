# Реестр слабых мест проекта/статьи «Россия 3.0 — Троица» (для усиления v3)

Анализ перед переработкой. Каждое слабое место → как закрываем в v3.

## Технические / научные
- **W1. Кремний не измерен.** Все цифры энергоэффективности — проекция; реальных данных с кристалла нет.
  → v3: сохранить честный тег [открытая гипотеза/проекция]; усилить тем, что FPGA-результаты [измерено] — это уже проверяемый якорь.
- **W2. Нет независимых цитирований/принятия индустрией GoldenFloat.**
  → v3: честно как [открытая гипотеза]; усилить через путь стандартизации (ПНСТ/ТК164) как механизм институционализации, а не «нас процитируют».
- **W3. Преимущество формата над posit/MX не доказано.**
  → v3: оставить как есть (не заявляем превосходства); усилить тезисом проверяемости, а не точности.

## Правовые (главные пробелы v2)
- **W4. РИД сведён только к авторскому праву (ст.1262) и топологии — НЕ упомянут патент.** Это недозащита: способ вычисления и устройство патентоспособны.
  → v3: новый §5.6 (патент способа/устройства, Приказ Минэк №148, льготные пошлины физлица; Apache 2.0 patent grant снимает мнимое противоречие).
- **W5. Авторство ИИ-кода не проговорено** — риск отказа в охране «машинного» кода.
  → v3: §5.5 (творческий вклад человека, ст.1257/1259 + Пленум ВС №10 п.80, практика 2024-26; рекомендация — журнал авторских решений).
- **W6. Экспортный контроль и зарубежное изготовление замолчаны** — главная честная напряжённость (Tiny Tapeout US = экспорт технологии).
  → v3: усилить §5.4 абзацем про ФЗ183-ФЗ + ПП1299; честно обозначить tension, не ослаблять.
- **W7. «Доверенный ИИ» в v2 опирался на законопроект как главный ориентир** — слабее, чем действующие нормы.
  → v3: добавить ПП№1931 (доверенное ПО, с 01.03.2026) и ПП№1912 (ПАК для КИИ) как ДЕЙСТВУЮЩИЕ нормы → перенести вес с законопроекта на действующее право.
- **W8. Криптофункция BLAKE3 (Euler) не квалифицирована** — риск по СКЗИ/категории 5 ТДН.
  → v3: §5.7 (ПП№313 + Решение ЕЭК №30: hash без ключа ≠ СКЗИ; keyed/MAC = СКЗИ → нотификация ЕЭК; для собственных нужд лицензия не нужна).

## Рыночные / организационные
- **W9. Юр.форма самозанятого блокирует большинство грантов и реестр ПО.**
  → v3: в дорожную карту явный этап «ИП/ООО → грант»; в обращении — просьба об облегчённом входе (уже было, усилить конкретикой грантов 2026).
- **W10. Нет механизма институционализации формата.**
  → v3: §4.1 + обращение — путь GoldenFloat → ПНСТ через ТК164 (нет ГОСТ на форматы чисел — окно).
- **W11. Гранты 2026 не названы конкретно** (Сколково приостановлен, НТИ-Нейронет ждать 12-ю очередь, Минпромторг ЭКБ до 350 млн).
  → v3: дорожная карта + обращение с актуальными программами и статусами.

## Риторические / стратегические
- **W12. Зарубежный IP-холдинг vs «всё отечественное» — противоречие, ослабляющее обращение.**
  → v3: усилить честный блок §5.4 — развести «проверяемость через открытые спецификации» (не зависит от юрисдикции) и «национальная приписка» (российское ООО-лицензиат + РИД в РФ); признать tension прямо.
- **W13. Обращение из 5 пунктов не закрывало патент/стандартизацию.**
  → v3: дополнить пункты обращения мерами по патентам open-hardware и по ускоренному ПНСТ форматов через ТК164.

# Второй проход (углубление, v3→v4) — стратегические/убедительные слабые места

## Самые важные новые пробелы
- **W14. НЕТ раздела «Риски и контраргументы».** Обращение к госуровню без честного разбора возражений выглядит наивно и снижает доверие рецензента ВАК. Это главный пробел.
  → v4: новый §9 «Риски, ограничения и ответы на контраргументы» (до Заключения): риск непринятия формата; риск «велосипеда» (зачем не IEEE754/posit); риск кадрового бутылочного горлышка (один автор); риск зарубежного изготовления/санкций; риск, что «проверяемость» воспримут как маркетинг.
- **W15. «Один автор» — риск устойчивости (bus factor).** Не проговорено, кто продолжит проект.
  → v4: в риски + дорожную карту — институционализация (вуз/партнёр, открытая модель контрибьюторов).
- **W16. Нет количественного определения «проверяемости».** Тезис verifiability-per-dollar силён, но не операционализирован → выглядит лозунгом.
  → v4: §3 — добавить конкретные измеримые прокси проверяемости (bit-exact conformance vectors, воспроизводимость seal/«печатей», 9/9 воспроизведение ширин, формальная спецификация .t27), чтобы метрика была технически измеримой, а не риторической.
- **W17. Конкурентный ландшафт не показан.** Не объяснено отличие от отечественных аналогов (Эльбрус, NPU Сбера/Яндекса) и зарубежных (posit/MX).
  → v4: краткий абзац позиционирования — Trinity не конкурент ускорителям, а ортогональный слой доверенной арифметики/оракула соответствия.
- **W18. Бизнес-устойчивость/финансирование не показаны.** Откуда деньги до грантов.
  → v4: в дорожную карту — bootstrap-этап (открытый код + сервисы), затем грант через ИП/ООО (уже есть), плюс лицензирование РИД.

## Закрываем в v4
- Новый §9 «Риски и контраргументы» (5-6 пунктов с честными ответами).
- §3: операционализация «проверяемости» (измеримые прокси).
- §4/позиционирование: ортогональность к ускорителям.
- Дорожная карта: bus factor + bootstrap-финансирование.

# Третий проход (углубление, v4→v5) — уровень рецензента ВАК / грантовой экспертизы

## Самые важные новые пробелы
- **W19. Нет явной научной новизны и положений, выносимых на рассмотрение.** Главный риск отклонения в журнале: не отделён новый результат от обзорно-публицистической части.
  → v5: блок во Введении — 3 элемента новизны (критерий verifiability-per-dollar; замкнутое правило GoldenFloat; метод честного фрейминга) + 3 положения П1–П3.
- **W20. Воспроизводимость без явного артефакт-протокола.** Рецензент потребует reproducibility-блок.
  → v5: §4.1 — протокол воспроизводимости (а–г): препринт, открытый код+хеш коммита+«печать», FPGA-тест-план, скрипт тождества Люка.
- **W21. Метрика verifiability-per-dollar не операционализирована в числах.** Прокси названы (W16), но нет примера расчёта.
  → v5: §3 — иллюстративная формула (доля воспроизводимых результатов / TCO), пример GF16 (35/35=1,0 на FPGA), пред-регистрируемый протокол.
- **W22. Конфликт «зарубежный IP-холдинг ↔ реестр ПО» не конкретизирован.** Новый юр.факт: ПП № 1236 — доля иностр. участия ≤ 49 %.
  → v5: §5.4 — факт ПП № 1236; вывод — совместимо только через РФ-ООО-лицензиат с контрольной российской долей.
- **W23. Обращение одностороннее (просьбы), нет встречной отдачи.**
  → v5: абзац «Встречная отдача государству» (4 актива: РИД, ПНСТ, кадры, доверенный компонент ФСТЭК) + контраргумент 7 в §9.

## Закрыто в v5 (14 стр.)
- Блок научной новизны/положений (Введение) — W19.
- Протокол воспроизводимости (§4.1) — W20.
- Иллюстративная операционализация метрики (§3) — W21.
- ПП № 1236 и ограничение ≤ 49 % (§5.4) — W22.
- «Встречная отдача» + контраргумент 7 — W23.
- Аннотация и список литературы обновлены.

# Четвёртый проход (углубление, v5→v6) — тонкий научный уровень

## Самые важные новые пробелы
- **W24. Нет раздела «Связанные работы» (related work).** Формальное требование ВАК-статьи: тезис и GoldenFloat не позиционированы относительно мировой литературы.
  → v6: новый §3a «Связанные работы» — posit (Густафсон 2017), takum (Хунхольд 2024), OCP MX/OFP8; BitNet b1.58; ZKML/TEE. Отличие GoldenFloat — проверяемость, не точность.
- **W25. Угроза самому тезису не разобрана.** «Проверяемость арифметики ≠ проверяемость модели» (галлюцинации, состязательные атаки выше арифметики).
  → v6: §3 — честная граница применимости: Trinity решает необходимую, но не достаточную часть; комплементарен ZKP/TEE.
- **W26. Нет численного ориентира для предлагаемого КПЭ «проверяемость».**
  → v6: §3 — иллюстративный baseline (2026 ≈ 0 %) → целевое (2030), метод сбора через реестр доверенного ПО + ФСТЭК.
- **W27. Международный/санкционный контекст односторонн** (только РФ-экспортконтроль).
  → v6: §5.4 — зеркальная иностранная сторона (US-ограничения на передовые техпроцессы/САПР); ответ — открытая публикация + зрелые узлы SKY130/GF180.
- **W28. Энергоэффективность троичности — только «49× проекция», без механизма.**
  → v6: §4.2 — физический механизм (троичные веса → сложение/вычитание вместо умножения, разрежённость нуля); механизм [доказано в литературе], цифра — открытая гипотеза.

## Закрыто в v6 (15 стр.)
- §3a «Связанные работы и позиционирование» — W24.
- Граница применимости тезиса (§3) — W25.
- Иллюстративный ориентир КПЭ (§3) — W26.
- Зеркальный экспортный контроль (§5.4) — W27.
- Физический механизм энергоэффективности (§4.2) — W28.
- Аннотация и список литературы (posit/takum/BitNet/ZKML) обновлены.
