# Реестр слабых мест проекта/статьи «Россия 3.0 — Троица» (для усиления v3)

Анализ перед переработкой. Каждое слабое место → как закрываем в v3.

## Технические / научные
- **W1. Кремний не измерен.** Все цифры энергоэффективности — проекция; реальных данных с кристалла нет.
  → v3: сохранить честный тег [открытая гипотеза/проекция]; усилить тем, что FPGA-результаты [измерено] — это уже проверяемый якорь.
- **W2. Нет независимых цитирований/принятия индустрией GoldenFloat.**
  → v3: честно как [открытая гипотеза]; усилить через путь стандартизации (ПНСТ/ТК164) как механизм институционализации, а не «нас процитируют».
- **W3. Преимущество формата над posit/MX не доказано.**
  → v3: оставить как есть (не заявляем превосходства); усилить тезисом проверяемости, а не точности.

## Правовые (главные пробелы v2)
- **W4. РИД сведён только к авторскому праву (ст.1262) и топологии — НЕ упомянут патент.** Это недозащита: способ вычисления и устройство патентоспособны.
  → v3: новый §5.6 (патент способа/устройства, Приказ Минэк №148, льготные пошлины физлица; Apache 2.0 patent grant снимает мнимое противоречие).
- **W5. Авторство ИИ-кода не проговорено** — риск отказа в охране «машинного» кода.
  → v3: §5.5 (творческий вклад человека, ст.1257/1259 + Пленум ВС №10 п.80, практика 2024-26; рекомендация — журнал авторских решений).
- **W6. Экспортный контроль и зарубежное изготовление замолчаны** — главная честная напряжённость (Tiny Tapeout US = экспорт технологии).
  → v3: усилить §5.4 абзацем про ФЗ183-ФЗ + ПП1299; честно обозначить tension, не ослаблять.
- **W7. «Доверенный ИИ» в v2 опирался на законопроект как главный ориентир** — слабее, чем действующие нормы.
  → v3: добавить ПП№1931 (доверенное ПО, с 01.03.2026) и ПП№1912 (ПАК для КИИ) как ДЕЙСТВУЮЩИЕ нормы → перенести вес с законопроекта на действующее право.
- **W8. Криптофункция BLAKE3 (Euler) не квалифицирована** — риск по СКЗИ/категории 5 ТДН.
  → v3: §5.7 (ПП№313 + Решение ЕЭК №30: hash без ключа ≠ СКЗИ; keyed/MAC = СКЗИ → нотификация ЕЭК; для собственных нужд лицензия не нужна).

## Рыночные / организационные
- **W9. Юр.форма самозанятого блокирует большинство грантов и реестр ПО.**
  → v3: в дорожную карту явный этап «ИП/ООО → грант»; в обращении — просьба об облегчённом входе (уже было, усилить конкретикой грантов 2026).
- **W10. Нет механизма институционализации формата.**
  → v3: §4.1 + обращение — путь GoldenFloat → ПНСТ через ТК164 (нет ГОСТ на форматы чисел — окно).
- **W11. Гранты 2026 не названы конкретно** (Сколково приостановлен, НТИ-Нейронет ждать 12-ю очередь, Минпромторг ЭКБ до 350 млн).
  → v3: дорожная карта + обращение с актуальными программами и статусами.

## Риторические / стратегические
- **W12. Зарубежный IP-холдинг vs «всё отечественное» — противоречие, ослабляющее обращение.**
  → v3: усилить честный блок §5.4 — развести «проверяемость через открытые спецификации» (не зависит от юрисдикции) и «национальная приписка» (российское ООО-лицензиат + РИД в РФ); признать tension прямо.
- **W13. Обращение из 5 пунктов не закрывало патент/стандартизацию.**
  → v3: дополнить пункты обращения мерами по патентам open-hardware и по ускоренному ПНСТ форматов через ТК164.
